# Release Checklist

Use this checklist when preparing a tagged release or a milestone update.

## Content

- [ ] Labs added or updated are linked from `labs/README.md`.
- [ ] English and Chinese docs are aligned for learner-facing changes.
- [ ] Examples have clear `README.md` files.
- [ ] `.env.example` files contain placeholders only.

## Quality

- [ ] Markdown lint passes.
- [ ] Link check passes.
- [ ] GitHub Pages deploy succeeds.
- [ ] No generated dependencies are committed.

## Community

- [ ] Issues are labeled.
- [ ] Good first issue candidates are current.
- [ ] Release notes mention new labs, examples, and docs.
