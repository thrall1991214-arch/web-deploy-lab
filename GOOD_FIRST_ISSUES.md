# Good First Issues

These are small, useful contribution ideas for new contributors.

## Documentation

- Add screenshots to Lab 01 after deploying the plain static example.
- Add a provider-specific note for GitHub Pages, Netlify, Vercel, or Cloudflare Pages.
- Improve one common failure explanation with a real error message.
- Translate a new English guide into Chinese, or update the Chinese version after an English change.

## Examples

- Add a `404.html` page to `examples/plain-static`.
- Add a smoke test note to one example README.
- Add a provider-specific output directory note to the Vite or React example.

## Quality

- Improve the link check ignore list if a provider blocks automated checks.
- Add a small Markdown style rule and document why it helps learners.

## How To Start

1. Pick one focused idea.
2. Open an issue and mention that you want to work on it.
3. Keep the pull request small.
4. Include how you verified the change.
