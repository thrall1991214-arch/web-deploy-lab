Project Goal

把部署变成可以反复练习的技能。

web-deploy-lab 面向已经掌握前端基础的开发者，把上线前后的关键工作拆成小实验： 生产构建、环境配置、预览环境、域名、HTTPS、日志和回滚思路。

Lab Catalog

每个实验只训练一个上线能力。

01

部署静态站点

练习生产构建产物、静态托管、缓存行为和上线后验证。

02

管理环境变量

区分公开变量、私密变量、构建时变量和运行时变量。

03

使用预览部署

用分支和 Pull Request 预览环境提前发现问题。

04

连接自定义域名

理解 DNS、HTTPS 证书、重定向和域名验证。

05

排查构建失败

阅读构建日志，定位依赖、环境和输出目录问题。

Learning Path

按照真实上线顺序推进。

  1. 准备 确认构建命令、输出目录、依赖版本和环境变量。
  2. 发布 部署到预览环境，完成冒烟测试，再进入生产环境。
  3. 验证 检查页面、网络请求、重定向、缓存和错误日志。
  4. 复盘 记录失败案例，把修复步骤沉淀成团队清单。
