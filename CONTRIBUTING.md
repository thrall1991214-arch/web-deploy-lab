# Contributing

Thanks for helping improve `web-deploy-lab`.

This project is meant to be practical, friendly, and useful for frontend developers who want more confidence deploying web projects.

## Ways To Contribute

- Add a hands-on lab
- Improve lab clarity
- Add troubleshooting notes
- Fix broken links or typos
- Suggest a better learning path

## Lab Guidelines

Good labs are small and focused. Prefer one deploy-related skill per lab instead of trying to cover an entire platform at once.

Each lab should include:

- A real scenario
- Clear learning goals
- Prerequisites
- Step-by-step tasks
- Verification steps
- Common failures and fixes
- Optional stretch exercises

Use [labs/README.md](labs/README.md) as the starting template.

## Pull Request Checklist

Before opening a pull request:

- Keep the change focused
- Check spelling and links
- Avoid committing secrets, tokens, or account-specific values
- Explain what changed and why
- Include screenshots or logs when they clarify deployment behavior

## Style

- Write for learners, not experts
- Prefer concrete steps over abstract advice
- Keep examples provider-neutral when possible
- When a lab depends on a provider, name the provider clearly
