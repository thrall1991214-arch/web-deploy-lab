# Troubleshooting Glossary

## 404

The server could not find the requested path. Check deployment root, routing fallback, and asset paths.

## Build Command

The command a provider runs to create deployable output, such as `npm run build`.

## Build Log

The provider output that shows install, build, and deploy steps. Look for the first meaningful error.

## Cache

Stored responses that can make old files appear after a deployment. Check browser, CDN, and provider cache behavior.

## DNS

Domain Name System records that point a domain to a hosting provider.

## Environment Variable

A named configuration value supplied outside source code.

## Output Directory

The folder that contains deployable files after build.

## Preview Deployment

A deployment created from a branch or pull request before merging to production.

## Smoke Test

A small set of checks that confirms a deployment is basically healthy.

## SSL/TLS Certificate

A certificate that enables HTTPS for a domain.
