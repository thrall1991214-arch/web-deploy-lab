# Beginner Learning Path

This path helps frontend developers practice deployment in a practical order.

## 1. Learn The Shape Of A Deployment

Start with Lab 01. Deploy a static site and verify the production URL.

Focus on:

- What files are deployed
- Which URL is production
- How to confirm assets load

## 2. Add Configuration

Move to Lab 02. Practice environment variables with placeholder values.

Focus on:

- Public values vs secrets
- Local vs preview vs production
- Build-time vs runtime configuration

## 3. Review Before Production

Use Lab 03. Create a branch, open a pull request, and inspect the preview deployment.

Focus on:

- Preview URLs
- Smoke tests
- Reviewer feedback

## 4. Make The Site Real

Use Lab 04. Connect a custom domain and verify HTTPS.

Focus on:

- DNS records
- HTTPS certificates
- Canonical redirects

## 5. Debug Like A Maintainer

Use Lab 05. Read a failed build log and document the recovery.

Focus on:

- First meaningful error
- Reproduction
- Smallest fix

## Suggested Weekly Rhythm

- Day 1: Read the lab and run locally.
- Day 2: Deploy a preview.
- Day 3: Verify and document failures.
- Day 4: Repeat with a small variation.
- Day 5: Write down what should become a checklist.
