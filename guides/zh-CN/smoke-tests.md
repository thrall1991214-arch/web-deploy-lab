# 部署冒烟测试

冒烟测试是一组快速检查，用来发现明显的部署问题。

## 基础检查

- 在隐私窗口打开生产 URL。
- 确认首页返回 HTTP 200。
- 确认 CSS 和 JavaScript 加载成功。
- 检查浏览器控制台是否有部署相关错误。
- 对页面执行硬刷新。

## 路由检查

- 打开首页。
- 打开一个二级路由。
- 刷新二级路由。
- 如果站点定义了 404 页面，检查一个不存在的路径。

## 数据检查

- 确认 API 请求使用预期环境。
- 如果有表单，提交一次安全测试数据。
- 确认公开环境标识正确。

## 域名检查

- 打开 `http://`。
- 打开 `https://`。
- 如果 apex 和 `www` 都存在，分别检查。
- 确认重定向落到规范域名。
