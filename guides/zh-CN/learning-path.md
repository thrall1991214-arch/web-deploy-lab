# 初学者学习路径

这条路径帮助前端开发者按照真实上线顺序练习部署。

## 1. 理解部署的形状

从 Lab 01 开始，部署一个静态站点并验证生产 URL。

重点关注：

- 哪些文件会被部署
- 哪个 URL 是生产环境
- 如何确认资源加载成功

## 2. 加入配置

继续 Lab 02，用占位值练习环境变量。

重点关注：

- 公开值和密钥的区别
- 本地、预览和生产环境的区别
- 构建时配置和运行时配置的区别

## 3. 生产前先评审

使用 Lab 03，创建分支、打开 Pull Request，并检查预览部署。

重点关注：

- 预览 URL
- 冒烟测试
- 评审反馈

## 4. 让站点变得真实

使用 Lab 04，连接自定义域名并验证 HTTPS。

重点关注：

- DNS 记录
- HTTPS 证书
- 规范域名重定向

## 5. 像维护者一样排查

使用 Lab 05，阅读失败构建日志并记录恢复过程。

重点关注：

- 第一个有意义的错误
- 本地复现
- 最小修复
